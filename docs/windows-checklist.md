# Windows 실장비 수동 검증 체크리스트

CI artifact `xr-touch-widget-windows`(NSIS `.exe` / MSI `.msi`)를 터치스크린 PC에 설치 후 확인.
spec §9 "최종 검증" 단계 — Mac/Linux에서 검증 불가한 항목만 모음.

## 설치
- [ ] Windows 버전 확인 후 기록 (10 / 11 — spec §12 미해결 항목 해소)
- [ ] WebView2 런타임 존재 확인 (Win11 기본 탑재; 없으면 installer 안내대로 설치)
- [ ] NSIS installer(`xr-touch-widget_x.y.z_x64-setup.exe`)로 설치 → 실행됨

## Glass / 창
- [ ] acrylic 재질이 실제로 보임 (창 뒤 콘텐츠가 블러되어 비침)
- [ ] Win11이라면: mica로 바꿔볼 가치가 있는지 육안 비교 메모 (현재 코드는 acrylic)
- [ ] 배경/버튼 투명도 슬라이더가 acrylic 위에서 의도대로 보이는지 (macOS vibrancy와 재질 다름 — 여기가 최종 사인오프)
- [ ] 둥근 모서리: macOS는 objc로 처리, Windows는 OS 기본 라운딩 — 모서리 아티팩트 없는지
- [ ] 팔레트가 데모 페이지·다른 앱 위에 항상 떠 있음 (always-on-top)
- [ ] 팔레트 창 밖 영역의 터치가 아래 앱에 정상 전달됨

## 터치
- [ ] 트리거 버튼 터치 정확도 (오터치·무반응 없음, flash 피드백 보임)
- [ ] ☰ 핸들 터치 드래그로 창 이동 (드래그 중 편집 모드 안 켜짐)
- [ ] ☰ 핸들 1초 정지 홀드 → 편집 모드 진입(accent 테두리 + SE grip), 짧은 탭 → 해제
- [ ] SE grip 터치 드래그로 크기 조절 → 재시작 후 크기 유지
- [ ] 설정 창의 input에 터치 키보드로 입력 가능
- [ ] 설정 창 상시 스크롤바를 터치로 스크롤 가능

## 왕복 (mock-xr 또는 실제 UE)
- [ ] 트리거가 수신됨 (mock-xr 로그 또는 UE 그래픽 재생)
- [ ] 상태점(설정 창의 active 장비 + 팔레트): 장비 켜짐=녹색, 끔=3초 내 빨강, 재기동=녹색 복귀
- [ ] 비활성 장비엔 ping 안 감(heartbeat 없음) — 상태점도 없음 (D13)
- [ ] 설정 변경(장비/버튼/외형) [적용] → 재시작 후 유지 / [뒤로가기] → 폐기·복원

## 견고성 (실장비 특유 — 최종 리뷰에서 코드로만 검증됨)
- [ ] pong 포트(8001) 선점 상태에서 실행 → 앱이 뜨고 경고 표시, 트리거는 정상 (CRIT-2 실기기 확인)
- [ ] 장비 IP를 일부러 오타(예: `192.168.0.10x`)로 저장 → 트리거 눌러도 팔레트 안 멈춤, 해당 장비 빨간 점 (CRIT-1 실기기 확인)
- [ ] 프로그램 종료: 설정 창 [프로그램 종료] 버튼으로만 종료됨 (팔레트엔 종료 경로 없음)

## 장기 상주 smoke
- [ ] 반나절 이상 켜둔 뒤: 메모리 증가 추세·상태점 정상·트리거 즉응성 유지
